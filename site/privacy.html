Privacy

Local-first by design

WebRoutines stores routines, runner state, and history in browser-managed local storage. No account is required, and no analytics/tracking is part of core usage.

What is stored

  • Routine definitions (names + ordered links)
  • Run history (start/stop/timing/progress)
  • Settings (theme, load mode, toggles)
  • Active session state (focused runner + step)

What is not done

  • No telemetry or third-party analytics
  • No account creation/auth requirement
  • No automatic cloud sync dependency
  • No ad/tracker script in extension runtime

Data lifecycle

  • Create/manage data from extension UI
  • Export/import backups manually as JSON
  • Delete routines/history from extension
  • Uninstall removes extension-managed data

Permissions rationale

Permission Reason
storage Persist routines, settings, and run data locally.
tabGroups Create and manage routine tab-group sessions.
unlimitedStorage Support larger local datasets over time.
host_permissions Enable focus mode overlay on pages; no page-content harvesting.
