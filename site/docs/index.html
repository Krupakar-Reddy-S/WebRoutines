Getting started

WebRoutines is a local-first extension for Chromium-based browsers that turns repeat browsing into ordered workflows.

  1. 1. Run bun install
  2. 2. Build with bun run build
  3. 3. Open chrome://extensions and enable Developer mode
  4. 4. Load unpacked extension from .output/chrome-mv3

Core idea

Define routines once, then run them daily with deterministic step navigation.

Interface

Use Sidepanel as primary control center, Popup for quick focused runner controls.

Section

What is a routine?

A routine is an ordered list of links you want to execute repeatedly as a single flow.

Contains

  • Routine name
  • Ordered links
  • Optional link metadata
  • Last-run markers

Purpose

  • Avoid opening the same tabs manually
  • Keep predictable sequence
  • Reduce context-switching overhead
Example routine: "Morning Ops"
1. dashboard.example.com
2. alerts.example.com
3. ci.example.com

Section

What is a runner?

A runner is an active routine session in a tab group. Multiple runners can be active; one runner is focused.

Focused runner controls

Previous Next Open current Stop

Tab group model

Each running routine maps to a Chrome tab group for clear separation.

Progress

Runner tracks step index, elapsed time, and completion state.

Section

Settings available

Settings are stored locally and apply to sidepanel + popup behavior.

Appearance

Theme: system, light, dark

Tab loading

Mode: eager (open all tabs) or lazy (open as you navigate)

Confirm before stop

Adds a stop confirmation prompt for safety.

Focus mini-controller

Enable or disable compact on-page runner controls.

Section

Keyboard shortcuts

Defaults control focused runner navigation.

Action Shortcut Behavior
Previous step Alt+Shift+Left Moves focused runner backward
Next step Alt+Shift+Right Moves focused runner forward

Change shortcut bindings in chrome://extensions/shortcuts.

Section

Local data model

WebRoutines is local-first. Core data stays inside your browser storage surfaces.

Stored locally

  • Routines
  • Run history
  • Settings

Not required

  • User accounts
  • Cloud sync
  • External analytics

Portability

  • JSON export backups
  • Import backups locally
  • User-controlled lifecycle
